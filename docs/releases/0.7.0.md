## DeepSeek++ 0.7.0

0.7.0 的主线不是增加几个独立入口，而是把 DeepSeek++ 从 DeepSeek 网页增强，推进到可以长期承接项目任务的浏览器内 AI Agent 工作台。这个版本补齐了持续工作最关键的三层能力：可复用的项目上下文、可下载的交付产物，以及可由用户审查和接管的工具执行过程。

### 主要变化

- **项目上下文成为任务入口**：可以把仓库、网页、本地文件和手工说明整理为项目资料库，并在对话时只带入和当前任务相关的片段，减少反复粘贴背景。
- **回答可以沉淀为文件产物**：新增单文件和项目包产物，适合保存脚本、Markdown、JSON、HTML、小型原型或文档集合，让 DeepSeek 的输出更容易进入真实工作流。
- **工具执行更适合人机协作**：沙箱确认、Skill 草稿、记忆导入预览、保存片段和提示词控制集中收口，用户可以在关键步骤前检查、修改或拒绝，而不是被动接受自动执行结果。
- **侧边栏更像工作台**：侧边栏对话、右键场景、官方 API Key 配置、工具结果恢复和页面导航继续打磨，刷新后也能保留更清晰的执行反馈。
- **会话和材料更容易复盘**：保存项、历史标签/搜索、代码块下载、单消息导出、图片附件清单等补齐了日常整理能力，适合把一次对话拆成后续可复用的资料。
- **平台与发布质量继续收口**：新增 Android WebView 开发者基线和资产暂存链路，但不把它包装成已发布移动端版本；发布前继续覆盖中英文、本地工具、MCP、自动化、多浏览器打包、manifest policy 和 release 资产校验。

### 验证

- `npm run ci:quality`：覆盖 workflow、生产依赖审计、提示词冻结、编译、Vitest、中英文校验、自动化、MCP / Shell / PoW smoke、多浏览器构建、manifest policy、打包和 release 资产校验。
- `npm run smoke:web`：覆盖联网搜索和工具续跑提示。
- `npm run build:android`：验证 Android WebView 资源构建与暂存；APK 构建和 Android 单测仍需要本机 JDK。
- `npm pack --dry-run --workspace packages/shell-host`：确认 Shell Host npm 包内容。
